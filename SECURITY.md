# Security Policy

## Reporting a Vulnerability

We use GitHub Private Vulnerability Reporting to handle security disclosures.

To report a vulnerability, please use the GitHub Security Advisory form:
[Report a vulnerability](../../security/advisories/new)

Do not open a public issue for security vulnerabilities.

## Disclosure Policy

We follow a 90-day coordinated disclosure window. After 90 days from the initial report,
vulnerabilities may be publicly disclosed regardless of patch status, unless an extension
is mutually agreed upon.

## Scope

Please report any security issues you find in this repository's code or configuration.

## Out of Scope

- Vulnerabilities in dependencies (report those to the relevant upstream project)
- Social engineering attacks
- Physical security issues

## No PGP Key

We do not use a PGP key for vulnerability reports. Please use GitHub Private Vulnerability
Reporting as described above.
